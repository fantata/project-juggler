# Triage — How to Prioritise Incoming Work

When Chris asks "what should I work on?" or seems scattered across projects:

## Priority Order
1. Client work with deadlines this week
2. Client work with money awaiting (invoiceable)
3. Active client work (money attached, no immediate deadline)
4. Corporate gig prep (if upcoming)
5. Revenue-generating personal projects (Quizzyfied, Fast Five)
6. Active speculative projects with momentum
7. Everything else

## Red Flags to Surface
- More than 3 speculative projects active → "You've got N speculative
  projects. Want to pause or kill any?"
- Any project with no next_action → "X has no next action"
- Any project not touched in 14+ days that isn't paused/waiting →
  "Should X be paused?"
- Money awaiting > 14 days → "Chase payment on X"
- Calendar conflict or double-booking

## When Chris Adds Something New
If he's adding a new speculative project while client work is pending,
don't block him — but mention it: "Worth noting you've got [client work]
outstanding. This going on the back burner or replacing something?"
